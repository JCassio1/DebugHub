# DebugHub

I dislike the debugger from VSCode. When working with monorepos, it is slowly to initiate them and you need to wait before starting your different application. This solves that and keeps it simple.

DebugHub helps you manage all your VS Code debug configurations in one place.

You can:

- See all debuggers in a single list
- Quickly see status (`🟢` running, `🔴` stopped)
- Start/stop one debugger from the row action
- Start all or stop all with top actions
- Select multiple debuggers and press `OK` to start or stop them
